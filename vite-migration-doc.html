Background

The Boostiny frontend application (boostiny-web-beta) is currently undergoing a migration from Webpack (Vue CLI) to Vite 7 as its build system. Vite is a modern, ES-module-native build tool that provides significantly faster development server startup (<1 second), near-instant hot module replacement (<100ms), and faster production builds (30–60 seconds vs. 3–5 minutes).

Before this migration can be fully deployed through the CI/CD pipeline, the following infrastructure-level changes are required on the Jenkins build environment.

Requested Actions

# Action Details Reason
1 Upgrade Jenkins Node.js version Update the Node.js runtime used by Jenkins to a supported version.

Minimum required: Node 20 — specifically v20.19.0+, which ships with npm 10.8.2.

Node 22 "Jod" (LTS, Active) — v22.12.0+ — is fully compatible and currently requested.
Node 24 "Krypton" (LTS, Active) — v24.0.0+ — is also compatible and recommended for a longer support horizon.

The current version should be replaced in the Jenkins tool configuration or the relevant Dockerfile / agent image. 		Vite 7 requires a minimum of Node.js 20.19.0+. Node 22 and Node 24 LTS both satisfy this requirement and ensure full compatibility with Vite's native ESM support, the sass-embedded compiler, and all updated dependencies in package.json.
2 Ensure npm is up to date npm is bundled automatically with Node.js — no separate installation is required.

Node 20 (v20.19.0+) ships with npm 10.8.2.
Node 22 (v22.22.2) ships with npm 10.9.7.
Node 24 (v24.15.0) ships with npm 11.12.1.

Confirm the pipeline does not pin or downgrade npm via npm install -g npm@<version> in any pipeline stage. 		Vite 7 and its dependency tree require npm 7 or higher (lockfile v3). npm bundled with Node 20 (npm 10.x), Node 22 (npm 10.x), or Node 24 (npm 11.x) fully satisfies this requirement and supports all workspace and peer-dependency resolution features used by the project.
3 Remove npm rebuild node-sass from the pipeline Remove any pipeline step that runs npm rebuild node-sass or installs node-sass as a build prerequisite. The project has migrated from node-sass to sass-embedded (a pure-JS Dart Sass implementation). node-sass is no longer a dependency, is incompatible with Node 22, and will cause the build to fail if the rebuild step is still present in the pipeline.

Impact if Not Applied

  • CI/CD builds will fail when the Vite migration branch is merged to staging/production.
  • node-sass does not support Node.js 18+ and will throw a native binding error.
  • Vite 7 requires Node.js 20.19.0+ — builds on older versions will error out at startup.
Note: These changes are required before the Vite migration branch (feature/CS-21789-progressive-vite-migration) is merged. No application code changes are required from the DevOps side beyond the three items listed above.

Acceptance Criteria

  • Jenkins pipeline runs with Node.js 20 (minimum v20.19.0), Node.js 22 (latest LTS v22.22.2), or Node.js 24 (latest LTS v24.15.0, recommended).
  • npm is automatically bundled with the chosen Node.js installation — npm 10.8.2 for Node 20, npm 10.9.7 for Node 22, or npm 11.12.1 for Node 24. The pipeline does not pin or override the bundled npm version.
  • No npm rebuild node-sass step exists in any stage of the pipeline.
  • npm install and npm run build complete successfully on the Vite migration branch.
